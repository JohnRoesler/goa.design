Code Generation

goa relies on code generation to alleviate the need for reflection or repetitive coding. The goagen tool analyzes the API design and produces various artifacts including code, documentation, tests or even your own output. The end result is application code that is clean and terse.

Design Based

The goa language allows writing self-explanatory code that describes the design of the API. The description focuses on listing the resources and the actions they support. The goa language is flexible and stays out of the way, you get to decide how your API should work.

Read more »

Powerful Request Context

Every API action handler in goa gets a context as first argument. The context implements the Context interface from google's context package. This means built-in support for deadlines and cancelation signals. The context also exposes the request and response states with action specific helper methods generated from the design.
